## Affected Pages/Routes
- HubSpot company record workspace tab — verify target-account gating, snapshot load, summary, signals, and all explicit state behavior
- Future account prep surface, if the dominant moment of use becomes pre-call prep instead of record view — verify that the chosen surface still exposes the same trust and action contract

## Key Interactions to Verify
- Open eligible target account and see one clear reason to care now
- Open ineligible account and see correct gated state
- Inspect evidence behind a surfaced signal
- Distinguish fresh, stale, degraded, empty, and low-confidence states
- Confirm whether the primary action is signal-first or person-first after wedge lock
- Validate that missing-data guidance supports action instead of replacing it

## Edge Cases
- Account has no useful signals
- Account has only weak or contradictory signals
- Snapshot exists but top evidence is stale
- Enrichment source is unavailable
- Transcript association is low confidence
- `hs_is_target_account` is missing or wrong
- User lacks permission to see transcript-derived or inferred evidence
- Local snapshot and HubSpot live state disagree

## Critical Paths
- Eligible account -> snapshot load -> top reason now -> evidence inspection -> user takes action
- Eligible account with degraded data -> warning state -> healthy evidence still usable -> user understands limits
- No credible evidence -> explicit “nothing credible to recommend” state instead of fabricated guidance
- Trust-sensitive transcript path -> low-confidence association suppressed or routed to review instead of promoted into seller-facing recommendation

## Product-Proof Validation Before Engineering
- 5-account workflow study across real accounts
- Trust review on top surfaced signals
- Repeat-use trigger validation
- Comparison of record-view vs pre-call-prep vs daily-briefing moment of use

## Minimum Coverage Requirements Once Repo Exists
- Unit tests for signal eligibility, ranking inputs, evidence suppression, freshness rules, and confidence display mapping
- Integration tests for snapshot assembly, degraded-source handling, low-confidence transcript behavior, and permission filtering
- E2E tests for eligible/ineligible states, empty/stale/degraded states, evidence drill-in, and first-action UX
- Security tests for role-based visibility, transcript retention boundaries, and unsafe external content suppression
